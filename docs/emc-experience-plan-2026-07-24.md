## 一、机制改进策略（飞轮 v5：先修测量，再谈迭代）

### M-A 信号修复（治 H5）——测得准
1. 词表收编：diagnose template 词表单一来源（消 E9 三处漂移）；**前置：扩 eval 用例冻结后动 prompt**。
2. 信号接通：ChatRequest/diagnose 回传链路补 template 字段（消 E10），意图断言恢复双信号。
3. 断言硬化：参数正确性断言接 expect*（消灭恒 pass）；意图断言去软（概念型也须有可判信号）；出口分布纳入采集。

### M-B 指标升级（治指标失真）——测得全
4. 报告升级为 EMC-SUM v1（链路/产物/状态三行 + 批级头部：pass 率、timeout 率、落图率、t_p50/p95、llm 调用数、误杀/漏判计数）。
5. 采延迟分段（诊断/agent/作答三段）与 LLM 调用数——延迟进指标体系，不再隐身。
6. 判定分列：系统错误 vs 断言误杀/漏判分开计数，迭代对象一目了然。

### M-C 闭环接通（治 H6）——能迭代
7. 投票落盘 + 失败三元组（prompt/期望/实际）结构化产出，进 backlog 文件。
8. 失败回流五环：失败例 → prompt 负例池 / Tool SOP 卡负例位 / few-shot 库 / golden set / eval 用例。
9. 回归 gate：改动后重跑基线批，pass 率不降且 p95 不涨才许合入。
10. golden set：episodes.jsonl 真实会话回放标注，用例分布向真实场景校准（治 M5）；抽样改类别分层（治 M6）。

## 二、体验优化路径（产品侧：三波次）

### 波次 P0 · 止血（治 H1 表层 + M1/M2/M3）
| # | 项 | 治 | 要点（机制级，不含实现） |
|---|-----|-----|--------------------------|
| P0-1 | 新话轮滚动复位 | M1 | 发送新问题即恢复跟随到底；话轮内上滑停跟保留 |
| P0-2 | srcId 上传去重 | M3 | 文件指纹三态：同指纹复用 / 同名异内容弹覆盖或并存 / 异名同内容提示关联 |
| P0-3 | 模型路由 + 分段预算 | H1 | diagnose/answer 默认 Flash，疑难升 Pro；单问总预算分段下发，超段降级（review 异步、跳过 revise），**保证必有回答** |
| P0-4 | 阶段进度时间线 + 取消 | M2 | 诊断→选工具→执行→绘图→作答逐步点亮；工具成图即上图（增量落图，不等最终回答）；可取消 |
| P0-5 | 词表收编 + 信号接通 | H5 | 与 M-A-1/2 同一项，飞轮与被测系统共享此改动 |

### 波次 P1 · 机制本体（治 H2/H3/H4）
| # | 项 | 治 | 要点 |
|---|-----|-----|------|
| P1-1 | Layer Manifest 管线 | H2 | 导入时一次性计算（字段三级管线：嗅探→字段字典→LLM 兜底按 srcId 缓存；语义标注；质量预检），全链路复用：grounding / diagnose data_plan / 参数校验 / 去重 |
| P1-2 | Tool SOP 卡 × 10 | H3 | 每工具：用途/适用尺度/输入契约/前置条件/参数契约/产出契约/失败模式/正例2+负例1；进 prompt 也进文档 |
| P1-3 | method→tool 确定性映射 | H3 | diagnose 出 method（方法闭集），代码映射表绑工具；单工具任务 plan-once-execute 全覆盖（0 中间 LLM 轮）；多步链才走 Pro plan |
| P1-4 | 成图范式 + 落图自检 | H4 | 每工具默认 paint/图例/命名/空结果处理；产出自动核（非空/范围合理/非重复 srcId），不过则标 partial——**裁定器侧消灭假完成** |
| P1-5 | 回答图层芯片 + EMC-SUM 页脚 | H4 | 结论挂可点击图层芯片；无图结论显式标"文本推断·未落图"；回答页脚 = 与测试报告同 schema 的工程摘要 |
| P1-6 | review 异步化 | H1 | 先出答案，审查标记后补 |

### 波次 P2 · 长期投资（治 H1/H2 结构性余量）
| # | 项 | 要点 |
|---|-----|------|
| P2-1 | Dataset Registry | dataset/slice/revision 三级寻址（复用现有切片键），EMC 以 `dataset@T2` 引用数据 |
| P2-2 | LLM 网关 | 超时/重试/模型阶梯/每调用延迟与 token 遥测落 jsonl——P0-3 的载体，遥测喂飞轮批级头部 |

| P2-5 | 本地意图路由 | 规则/小模型分流 纯问答/单工具/多步骤，进一步降 LLM 调用数 |

## 三、执行步骤（批次 / 依赖 / 验收标准）

```
Batch 0（前置·护承重）→ Batch 1（P0 止血）→ Batch 2（飞轮 v5）→ Batch 3（P1 本体）→ Batch 4（P2 投资）
                ↘ 共享项：词表收编+信号接通（B1/B2 各做一次，不重复）
```

| 批次 | 内容 | 依赖 | 验收标准（DoD） | 量级 |
|------|------|------|------------------|------|
| **B0** | eval 扩例冻结（补 compare/filter_attr/负例期望）→ 词表收编（M-A-1）→ 信号接通（M-A-2） | 无 | eval 全绿；飞轮意图例 tmpl 列出现非 `?` 值 | 1-2 天 |
| **B1** | P0-1 滚动 / P0-2 去重 / P0-3 路由+预算 / P0-4 进度+取消 | B0（P0-3 依赖词表稳定） | 四痛点表层症状消除：新话轮自动到底；重复上传不堆叠；单工具任务 p95 < 60s（阶段目标）；全程有进度可取消 | 3-5 天 |
| **B2** | M-A-3 断言硬化 + M-B 报告升级 + M-C-7/8/9 闭环三件套 | B0 | 报告含 EMC-SUM v1+批级头；参数断言不再恒 pass；失败例自动入 backlog+负例池；回归 gate 可跑 | 2-3 天 |
| **B3** | P1-1 manifest → P1-2/3 SOP 卡+映射 → P1-4 范式+自检 → P1-5 芯片+页脚 → P1-6 异步 review | B1（预算制）+ B2（断言/gate） | 10 工具逐个过"三件套"（0 中间轮+范式+自检）；上传识别率 ≥95%；10 工具一次成功率 ≥90%；假完成率 = 0（gate 拦截） | 8-12 天 |
| **B4** | P2-1 Registry → P2-2 网关 → P2-3 异步任务 → P2-4 迭代引擎 → P2-5 路由 | B3 | 北极星全项达标（§〇表） | 10-15 天 |

**执行红线**：
1. diagnose prompt / 四态出口 / harness 主循环 = 承重：必须先扩 eval 再动，且每次只改一处（AGENTS.md 严格 SOP）。
2. P1-4 的 partial 判定改动触出口裁定 → 安排在 B3 且单独成 PR，附追踪 ID 清单。
3. 每批结束跑飞轮全量 + episodes 回放对比，指标进看板（before/after），形成下一批输入。
4. 不派 subagent；只 commit 不 push；报告纳入 git。
