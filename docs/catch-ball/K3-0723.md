## 一、任务定义

系统性审查 EMC（EmotionMap Copilot，含 CPD 情境式渐进披露引导）在**实际使用中**的工作策略与执行机制，评估其能否有效达成预期核心目标，明确指出当前机制中的**偏差、冗余与效能瓶颈**，并输出具体优化工作清单（需调整的策略方向 / 需补充的功能模块 / 需剔除的无效流程），确保机制与核心目标精准对齐。

## 二、用户已确认的审查约束（4 项）

| 项 | 用户确认结果 |
|---|---|

## 三、审查对象与现状事实（静态预读已核实）

### 3.1 EMC 链路（实际运行机制）

```
用户提问 → DIAGNOSE 认知层（6 字段卡：domain_lens/scale/decision_type/outlet/data_plan/method
           + 数据自检三态 ready / fallback_annotated / request_upload）
        → harness.orchestrate 编排（_quickIntent 本地预判短路
           / plan-once-execute（runTemplatePath 单技能 0 中间 LLM 轮）
           / ReAct while-loop 兜底，轮次上限已演进为 intent-aware——以代码为准）
        → answer（Pro 流式）→ Review（Flash 七条 checklist，同步阻塞）→ Revise ≤1 轮
        → 四态出口 result / gap / partial / ask / drift + general 短路（无 exit）
```

- **出口裁定**：EXIT_RESULT 条件 = intent∈{B,C} 且 (successObs>0 或 newLayerCount>0)——**部分成功也判"做成了"**，疑似痛点 4"假完成"根因区。
- **抗漂移三防线**：parseAgentStep 归一 + narrated 修复 ≤1 轮 + onDegraded 永不裸输。
- **GIS 骨干**：`api/geo_routes.py` 10 工具（filter_attr/clip/merge/area_stats/zonal_stats/rank/buffer/overlay/nearest/hotspot + /geo/density）+ `core/geo_registry.py` lazy 缓存；前端 `tools.js` SKILL_DEFS 参数契约 + validateParams——疑似痛点 3 根因区。
- **上传数据通路**：dropzone + openImport(sidebar.js) + import-input + range tab；hasImport 谓词（cpd-state.js，非 AI 组非 tool 产出 point 层）；grounding `buildContext`（tools.js:455-458）注入 diagnose context——疑似痛点 2 根因区。
- **耗时结构**：diagnose(Pro reasoner 流式) + agentStep N 轮 + answer(Pro 流式) + review(Flash 非流式同步阻塞) + revise(Pro ≤1 轮)——疑似痛点 1 根因区。

## 四、痛点 → 疑似根因区映射（实测设计基准，非结论）

| # | 用户痛点（逐字） | 疑似根因区（待实测+代码核实） |
|---|---|---|
| 1 | 效率很低，回答很慢 | 多 LLM 轮串行（diagnose Pro + agentStep N 轮 + answer Pro + review Flash 阻塞 + revise Pro）；Pro reasoner 本身慢；每轮 buildContext 体积；SSE 开销；**历年叠加的守卫/gate（_hardFail/_verifyClaims/narration 通道/ask_user/partial 等）是否冗余叠加** |

| 3 | 基本gis操作经常出错 | SKILL_DEFS slots/validateParams；geo_routes 端点参数契约；入参别名规整；图层名解析（boundary-resolve.js）；/geo 调用 status 失败模式 |
| 4 | 略复杂流程只做一半就停下或报告已完成 | ReAct 轮次上限打满即停；EXIT_RESULT 裁定 successObs>0 即报 result（部分成功假完成）；narrated 修复 ≤1 轮后直接 GAP；plan-once-execute 模板路径覆盖不足 |

## 九、承重边界与 Blast Radius

- **审查不改任何业务代码**（仅新增审计脚本 + 交付文档）；新增脚本放 `tests/browser/`（标准 pytest `--ignore=tests/browser` 跑法不受影响）。
- **承重三件套零触**：diagnose prompt 永不动（保 tests/eval_template_flash.py eval）/ 四态出口不动 / harness orchestrate 主循环不动。优化清单中涉及者显式标"需用户拍板"。
- **实测成本与稳定性**：串行发问 + 每问间隔防 429；DeepSeek 慢响应设重试与超时上限；headless；不起并发。
- **隐私脱敏**：episodes.jsonl 含用户原始问题（gitignore 已排除），报告引用一律脱敏（铁律 7，不含用户名/ID）。
- **只 commit 不 push**；**不派 subagent**（全局承重）。
- **证据可复现**：实测脚本留痕（问题集 + 原始记录 JSON），报告结论可重跑验证。
