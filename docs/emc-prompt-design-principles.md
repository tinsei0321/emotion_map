# EMC 测试用例 Prompt 设计原则（训练素材）

> 2026-07-23 · 用户定 · 测试飞轮 prompt 质量基准
> 每条测试用例的问题必须符合以下原则，否则"分析缺乏基准，失去意义"。

---

## 一、七大原则

### 1. 要素信息（基准）
提问必须包含**基本要素**才能展开分析：
- **范围**（行政区划/片区/用地类型/自定义范围）——分析的空间边界。
- **点数据**（情绪点/POI/设施）——分析的对象数据。
- **具体要求**（排序/归因/对比/分布/热点）——分析的产出类型。

**反模式**：「西陵区情绪归因」——缺具体要求（什么维度的归因？设施？环境？服务？）。
**正模式**：「西陵区范围内情绪点的 4×5 归因——哪些 domain×element 组合最消极？」

### 2. 拓扑关系（空间逻辑）
不能凭空分析点或范围的内容，必须基于**起码的拓扑关系**：
- **包含**：「X 范围**内**的点」
- **邻接/周边**：「X 设施**周边** N 米的点」
- **交集**：「X 范围与 Y 用地的**交集**」
- **对比**：「A 区**与** B 区的对比」

**反模式**：「哪些区域情绪最差」——无范围/无拓扑（哪些区域的什么？）。
**正模式**：「西陵区和伍家岗区范围内，哪个区的消极情绪占比更高？」

### 3. 目的性（精准意图）
要有**明确、精准的目的**，不是泛泛模糊的提问：
- 排序找极值（最好/最差 Top N）
- 归因找原因（为什么消极？4×5 哪格？）
- 分布看趋势（空间聚集？热点在哪？）
- 对比找差异（A vs B 差在哪？）
- 演变看时序（T1→T3 好转/恶化？）

**反模式**：「情绪分析」（目的不明——排序？归因？分布？）。
**正模式**：「西陵区范围内情绪最差 Top 3 片区，每个片区的 4×5 归因是什么？」

### 4. 颗粒度匹配（尺度诚实）
提问颗粒度须与数据支撑匹配（CLAUDE.md「产品本质」宏观护城河）：
- **宏观可答**：区域排序/分布趋势/主题倾向/对比差异。
- **微观不可答**：单条街精确情绪分/单栋楼评价/个人情绪追踪。

**反模式**：「夷陵广场某条街的情绪精确分是多少？」（微观·数据不足）。
**正模式**：「夷陵广场周边 500 米的宏观情绪倾向如何？」

### 5. 时态维度（演变视角）
利用 T1-T3 全局时间轴，部分提问应涉及时序演变：
- 「西陵区 T1→T3 情绪变化趋势如何？」
- 「哪片区域从 T1 消极到 T3 好转了？」

**非必须**（部分用例不涉及时序），但时间轴相关用例**必须**有明确时态要求。

### 6. 对比性（差异洞察）
对比是规划决策的核心——很多有价值的分析是比较性的：
- 空间对比（A 区 vs B 区）
- 时序对比（T1 vs T3）
- 类型对比（商业用地 vs 居住用地的情绪差异）

对比类提问须**至少 2 个明确的比较对象**。

### 7. 可操作性（规划价值）
提问应导向**对规划决策有用的洞察**（CLAUDE.md「演示逻辑链」）：
- 定位关注区（哪片最需要关注？）
- 主题倾向（主要矛盾是环境还是服务？）
- 排序优先级（先治哪片？）

**反模式**：「展示情绪地图」（仅显示，无分析价值）。
**正模式**：「西陵区情绪最差 Top 3 片区+主要矛盾（domain×element），为优先治理排序。」

### 8. 范围一致性（数据边界）
提问的范围必须**在已载范围内**——围绕**中心城区行政区划**及内部各行政区（西陵区/伍家岗区/夷陵区/点军区/猇亭区）+ **大南门·二马路滨江片区**展开。不得引用范围外或无法定位的地名（如「滨江绿地」「奥体中心」——无法判断哪里是"滨江"），除非已上传对应 POI/范围数据。

**反模式**：「滨江公园周边 500 米的情绪」（"滨江公园"位置不明确，范围数据中无此 POI）。
**正模式**：「大南门·二马路滨江片区范围内情绪最差 Top 3 片区的 4×5 归因」（范围=大南门·二马路滨江片区，已上传边界数据）。

> **POI 缓冲扩展**：除行政范围外，已载入的情绪点本身也可作为缓冲中心——「以数据中**某 POI 点**为中心，周边 500m 范围内情绪分布」。POI 名称须来自已载数据（如 CSV 中的 `spatial_hotspot` / `area_tag` 字段值），不得臆造。这扩展了分析类型：**范围聚合**（行政区/片区内）+ **POI 缓冲**（某设施/地点周边 N 米）+ **范围对比**（A 区 vs B 区）。

## 二、Prompt 结构模板

```
[范围] + [拓扑关系] + [点/设施数据] + [分析目的] + [可选：时态/对比维度]
```

**示例**：
- 「**西陵区范围内**（范围）的**情绪点**（数据），按**极性排序**找出**最差 Top 5**（目的），给出每个区域的 **4×5 归因**（深化目的）。」
- 「**滨江公园周边 500 米**（范围+拓扑）的**情绪点**（数据），与**奥体中心周边 500 米**（对比对象）相比，哪个区域**消极情绪占比更高**（对比目的）？」
- 「**西陵区与伍家岗区**（范围+对比）在 **T1→T3** 时段（时态），**情绪指数如何演变**（演变目的）？」

---

## 三、测试用例 prompt 检查清单

每条 LLM 用例的 `q` 字段发布前自检：

- [ ] 有明确范围（区/片/周边 N 米）？
- [ ] 有拓扑关系（内/周边/交集/对比）？
- [ ] 有精准目的（排序/归因/分布/对比/演变）？
- [ ] 颗粒度匹配（宏观可答·非微观精确）？
- [ ] 数据已加载（CSV 点层 + 边界范围）？
- [ ] 可验证（badge/layer/answer 可断言）？
