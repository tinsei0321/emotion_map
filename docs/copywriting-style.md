# 产品文本风格 (Copywriting Style)

> 写产品文案、按钮文字、提示语时必读本文档。

## UI 标记系统

**禁止使用 emoji。** 全部使用 ASCII 标记：

| 标记 | 含义 | 使用场景 |
|------|------|----------|
| `[OK]` | 成功 | 操作完成、数据保存 |
| `[WARN]` | 警告 | 潜在问题、需关注 |
| `[LOAD]` | 加载中 | 数据处理中、等待 |
| `[ERR]` | 错误 | 操作失败、异常 |

## 按钮文字风格

- 简短：2-4 字为宜
- 动词开头：`开始分析`、`在地图上显示`
- 状态双态切换：如 `开始分析` ↔ `在地图上显示`
- 中文优先（不做国际化可暂不考虑英文）

## 页面标题

- 格式：`情绪地图 — 子页面名`
- 侧边栏返回链接：`[返回地图浏览器](/)`

## 分析报告文本

- 用户为城市规划专业人员，用词可偏专业（"极性分布"、"空间自相关"、"缓冲区分析"）
- 数据指标使用中文标签，保留原始数字精度
- 避免"可能"、"大概"等模糊词（数据驱动，不猜测）

## 错误信息

- 格式：`[ERR] 具体原因 — 建议操作`
- 例：`[ERR] 数据文件未找到 — 请检查 DATA/raw/ 目录`
- 不要只写"出错"（无意义）
- 不要暴露内部路径给用户

## 术语表

| 术语 | 说明 |
|------|------|
| 情绪得分 (score) | SnowNLP 输出的 0~1 情绪分数 |
| 五级极性 | Very Negative / Negative / Neutral / Positive / Very Positive |
| 治理 (governance) | L1 数据清洗：坐标转换 + 相关性过滤 + 脱敏 |
| 管道 (pipeline) | L0→L1→L2→L3→L4 四级数据处理链路 |
| 范围 (range) | 用户上传的分析边界 Polygon |
