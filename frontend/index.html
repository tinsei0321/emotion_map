关于 · 宜昌市情绪地图
宜昌市情绪地图
prototype v0.1.0
基于多源社交数据（大众点评 / 美团 / 小红书 / 微博 / 12345 热线）的城市情绪空间分析平台 — 用数据替代直觉，用地图承载叙事。
数据管线
L0原始采集Scrapy 爬虫
→
L1数据治理坐标 + 脱敏
→
L2情绪极性SnowNLP
→
L3语义增强DeepSeek LLM
→
L4多维归因空间统计
项目架构 — 七层功能（数据自底向上流转）
应用层frontend/ MapLibre 主界面 + apps/ Streamlit 遗留 — 地图 / 图层 / 导入 / 分析控制台
UI 组件层core/ui_components HUD 按钮 / 图例叠加 / 标题栏（Streamlit 遗留，前端已迁 MapLibre）
空间分析引擎core/map_engine 天地图底图 / 点状情绪标记 / 热力图 / 缓冲区分析 / 行政聚合
数据分析引擎SCRIPT/ L0→L1 治理 / L1→L2 SnowNLP 情绪 / L3 DeepSeek 语义 / L4 归因 — 统一 run_analysis_task()
基础设施层core/ config 配置 / data_loader 加载 / export 导出 / tracker 决策追踪
数据采集层SCRAPER/ Scrapy 2.16 多源爬虫 — 大众点评 / 美团 / 小红书 / 微博 / 12345 热线
数据层DATA/ raw 原始 / processed L1~L4 治理+分析结果 / boundaries 边界范围
技术架构 — 代码目录
frontend/ MapLibre 主界面 core/ 基础设施 SCRIPT/ 分析管道 SCRAPER/ 数据采集 DATA/ 数据存储 design/ 设计令牌 docs/ 文档体系 apps/ Streamlit 遗留
技术栈
Python 3.14 MapLibre GL JS 5.x SnowNLP + jieba Shapely + proj4 shpjs (Shapefile) csv2geojson FastAPI Scrapy 2.16 CARTO 瓦片
应用场景 — 4 类需求 × 5 要素
设施环境服务文化事件
城市规划设施选址评估绿地/水系感知—文化设施布局—
城市更新老旧设施改造环境品质提升公共服务优化——
城市运营设施运维监测—服务效率评估—突发事件响应
城市治理设施投诉热点环境问题定位服务质量追踪文化活动效果舆情事件预警
5 要素是市民情绪的核心指向：设施（交通/教育/医疗）、环境（绿化/空气/噪音）、服务（政务/商业/公共）、文化（历史/休闲/体育）、事件（施工/事故/活动）。

社交媒体 GCJ-02 → WGS84 EPSG:4324（统一基准）→ CGCS2000 EPSG:4546（宜昌投影）。导入时自动读 .prj → proj4 重投影。
