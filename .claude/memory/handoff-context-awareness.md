# 交接任务的场景感知

交班时不要机械地认为"交接 = 第二天"。要根据时间段和上下文推断实际场景：

| 时间段 | 用户说 | 实际场景 |
|--------|--------|----------|
| 傍晚/晚上 | "收工" / "下班交接" | 办公室 → 家里（当天晚些时候换机） |
| 傍晚/晚上 | "到家了，交接任务" | 刚到家，**同一天**换机，不是第二天 |
| 早上/上午 | "交接任务" | 家里 → 办公室（第二天上班） |
| 任何时间 | "换机了" | 立即切换机器 |

**关键判断依据**：
1. 看实际时间，不是机械+1天
2. "到家了" = 同一天晚上换机，上下文应该是当天工作的延续
3. "第二天"的情况通常是早上到公司后的交接
4. session-handoff.md 中的"待办 06-16"在我晚上到家时还是未来，不应标记为"今天待办"
5. 交接时先拉取最新 git 状态，对比 handoff 文件中的记录，更新不准确的信息

**Why:** 用户晚上到家后继续工作，我错误地理解为"第二天"并标记了错误的日期，显得不够智能。
**How to apply:** 交接时先看当前时间 + 用户措辞，推断是同天换机还是跨天换机。更新 handoff 文件中的过时信息（如已提交的 commit hash、已不存在的风险提示）。
